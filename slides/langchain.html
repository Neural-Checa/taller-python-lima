Por qué existe

El problema que resuelve

Hablarle a un LLM "a mano" se vuelve complicado apenas quieres algo más que un chat simple:

  • Cada proveedor, su propia API — código distinto para OpenAI, Groq, Gemini…
  • Cambiar de modelo = reescribir tu aplicación entera.
  • Tareas comunes, hechas a mano — memoria de chat, usar herramientas, leer documentos… una y otra vez.

LangChain pone orden en todo eso, con piezas estándar y reutilizables.

El qué

¿Qué es LangChain?

Una librería de Python que estandariza cómo hablas con cualquier LLM y te da piezas listas para construir aplicaciones de IA.

Idea 1

Una interfaz común

El mismo código para cualquier modelo, de cualquier proveedor.

Idea 2

Bloques que encajan

Prompts, herramientas, memoria, agentes… listos para armar.

Beneficio 1

Un código, cualquier modelo

from langchain_groq import ChatGroq

llm = ChatGroq(model="llama-3.3-70b-versatile")

# ¿Cambiar de proveedor? Es UNA línea:
# from langchain_openai import ChatOpenAI
# llm = ChatOpenAI(model="gpt-...")

respuesta = llm.invoke("Hola, ¿quién eres?")

Mismo .invoke(), mismo código alrededor. Pruebas y cambias de modelo sin reescribir tu app.

Agentes

El modelo decide qué pieza usar y cuándo.

Las conectas como piezas de Lego para construir justo lo que necesitas.

Beneficio 3 · lo que haremos hoy

De "chatear" a agentes que actúan

  • Le das herramientas al modelo — consultar una base de datos, llamar una API, buscar en la web.
  • El modelo decide cuál usar y cuándo — tú no programas el flujo paso a paso.
  • LangChain + LangGraph arman el bucle por ti: pensar → actuar → observar → repetir.

Eso es exprimir el modelo: ya no solo responde, ACTÚA. Justo lo que haremos hoy con la base de datos.

En resumen

LangChain es el pegamento

Tú piensas en qué construir; LangChain se encarga del cómo hablarle a cada modelo y de las piezas comunes.

El combo que usaremos: LangChain (las piezas) · LangGraph (orquesta el agente) · LangSmith (para depurar y observar).
